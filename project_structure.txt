# SOURCE OF TRUTH: MICROSOFT OUTLOOK SECURITY CENTER
# VERSION: 2.3.0 (Hardened Release)
# TARGET: AI AGENT / SENIOR DEVELOPER HANDOFF

## 1. PROJECT MISSION
To provide a client-side, non-blocking "Security Sidecar" for Microsoft Outlook that automates threat isolation while preserving privacy. It employs zero-disk forensic analysis and hardware-backed credential protection to serve users in high-risk environments without requiring Exchange Admin rights.

## 2. HARDENED ARCHITECTURE (REALITY)
- **UI:** Electron.js (Context-Isolated, Node-Integration Disabled).
- **UI Architecture:** Component-Based Modular Architecture. Logic is segmented into specialized ui_*.js modules for maintainability.
- **Backend IPC:** Bidirectional Named Pipe with ENV-backed handshake. High-speed caching layer in the main process eliminates UI bottlenecks.
- **Dual-Store Database:** Optimized split storage. `config.json` for lightweight settings and `data.json` for high-volume historical scan records and processed IDs.
- **Engine:** 8-Core Parallel PowerShell Runspace Pool. Decouples Outlook COM extraction (STA) from heavy security analysis (MTA).
- **Credential Protection:** Windows DPAPI (via Electron safeStorage). API keys never exist in plaintext on disk and are decrypted only for active API sessions or user-requested exports.
- **Deduplication:** Content-aware SHA256 "Digital Fingerprints" (Sender + Subject + Timestamp) ensure that emails are tracked uniquely even if moved, copied, or re-scanned.

## 3. ZERO-DISK FORENSIC PROTOCOL
- **MIME Isolation:** Attachments and headers are streamed directly from the COM layer into memory.
- **Verified Transactional Moves:** No-split-brain logic. UI only updates after the backend performs a real-time probe to confirm an email has successfully landed in its target Outlook folder.
- **State Preservation:** Strictly non-disruptive analysis. Original UnRead/Read states are captured and force-restored after every scan or folder transition.
- **Deep-Chain IP Analysis:** Bottom-up parsing of Received headers that filters out known cloud relays (AWS/M365) to identify the true originating server.

## 4. RESOURCE & STABILITY MANAGEMENT
- **COM Lifecycle:** Strict `Marshal.ReleaseComObject` protocol for every reference to prevent `OUTLOOK.EXE` zombie processes.
- **Nuclear Reset Protocol:** Comprehensive system wipe that physically unlinks all database files, purges forensic JSON fragments, deletes security logs, and restores factory defaults.
- **Watchdog Engine:** 60-second hard-kill for hung scanning processes and real-time heartbeat monitoring.
- **Proactive Outlook Management:** Automated registry-based lookup and background launch of Outlook if not running.

## 5. SCANNING & PRIORITIZATION
- **OnAccess:** Real-time monitoring of incoming items.
- **Bulk Action Logic:** Marking an email as Safe/Spam automatically identifies and processes all other items in the current list meeting the same criteria (Sender, Domain, IP, or Combo).
- **ANTI-SPAM ENGINES (8-Tier Analysis):** 
    1. VirusTotal API (IP/URL Reputation).
    2. DMARC/SPF/DKIM Authentication.
    3. Sender/Domain Alignment.
    4. Reverse DNS Verification.
    5. Global RBL Blacklists.
    6. Anti-Phishing Shield (Body/Link Analysis).
    7. Smart Heuristics (Keyword Matching).
    8. IP|Domain Combo Filtering.
- **Portability:** Professional JSON Import/Export for the entire security configuration, including decrypted API keys for cross-system migration.

## 6. VERDICT ROUTING
- **MALICIOUS:** Confirmed Malware/Phishing -> **Deleted Items**.
- **SPAM:** High-score suspicion -> **Junk Email**.
- **SUSPICIOUS:** Moderate score -> UI Dashboard listing.
- **SAFE:** Whitelisted or Low-score -> **Inbox**.
