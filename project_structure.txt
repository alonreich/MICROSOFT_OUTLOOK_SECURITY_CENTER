# SOURCE OF TRUTH: MICROSOFT OUTLOOK SECURITY CENTER
# VERSION: 2.1.0 (Enterprise-Grade)
# TARGET: AI AGENT / SENIOR DEVELOPER HANDOFF

## 1. PROJECT MISSION
To provide a client-side, non-blocking "Security Sidecar" for Microsoft Outlook that automates threat isolation while preserving privacy. It employs zero-disk forensic analysis and hardware-backed credential protection to serve users in high-risk environments without requiring Exchange Admin rights.

## 2. HARDENED ARCHITECTURE (REALITY)
- **UI:** Electron.js (Context-Isolated, Node-Integration Disabled).
- **Backend IPC:** Bidirectional Named Pipe with ENV-backed handshake. Eliminates stdin buffering issues and race conditions.
- **Engine:** 8-Core Parallel PowerShell Runspace Pool. Decouples Outlook COM extraction (STA) from heavy security analysis (MTA).
- **Credential Protection:** Windows DPAPI (via Electron safeStorage). API keys never exist in plaintext on disk.
- **Database:** Atomic Electron-store with global file-locking and Read-Modify-Write transaction logic to prevent data loss during high-volume parallel scans.
- **Multi-Account Support:** Store-aware logic dynamically targets the specific Junk/Deleted folders of the originating account across multiple Outlook data files.

## 3. ZERO-DISK FORENSIC PROTOCOL
- **MIME Isolation:** Attachments are streamed directly from the COM layer into RAM byte arrays for hashing (SHA256/MD5).
- **Heuristic Reconstruction:** Automated synthetic header generation for internal Exchange/M365 items lacking standard SMTP transport headers.
- **Aggressive Body Parsing:** Tag-stripping HTML intercept for forensic view when plain-text body properties are empty.

## 4. RESOURCE & STABILITY MANAGEMENT
- **COM Lifecycle:** Strict `Marshal.ReleaseComObject` protocol for every reference to prevent `OUTLOOK.EXE` zombie processes.
- **Atomic State Clearing:** Hard reset uses file-lock synchronization to prevent runtime errors during configuration wipes.
- **Watchdog Engine:** 60-minute hard-kill for hung scanning processes and real-time heartbeat monitoring.

## 5. SCANNING & PRIORITIZATION
- **OnAccess:** Monitors [Unread] items globally across all folders and stores.
- **Automatic Cascade Release:** Restoring/Whitelisting an item automatically triggers a cross-category search and release for all matching items (IP, Sender, Domain, or Combination).
- **3-Tier Failover:** 
    1. VirusTotal API (Primary).
    2. Team Cymru MHR (DNS-based fallback).
    3. Local Heuristics/Spam Rubrics (Final).

## 6. VERDICT ROUTING
- **MALICIOUS:** Confirmed Malware/Phishing -> **Deleted Items** (Isolation).
- **SPAM:** High-score suspicion -> **Junk Email** (Quarantine).
- **SUSPICIOUS:** Moderate score or Reputation Hook -> UI Dashboard listing.
- **SAFE:** Whitelisted or Low-score -> **Inbox** (Untouched).
- **Restoration:** Perfect state recovery (Return to original folder with original Read/Unread status).

## 7. PROJECT FILE MANIFEST
- `main.js`: System core, IPC coordination, and atomic database management.
- `outlook-scanner.ps1`: Multi-threaded analysis engine and Outlook bridge.
- `index.html`: Forensic dashboard and security configuration UI.
- `preload.js`: Secure bridge between UI and Node.js.
- `package.json`: Dependency and build configuration.
- `silent_launcher.vbs`: Hidden startup coordinator.
- `MICROSOFT_OUTLOOK_SECURITY_CENTER.bat`: CLI entry point.
