# SOURCE OF TRUTH: MICROSOFT OUTLOOK SECURITY CENTER
# VERSION: 2.0.0 (Hardened)
# TARGET: AI AGENT / SENIOR DEVELOPER HANDOFF

## 1. PROJECT MISSION
To provide a client-side, non-blocking "Security Sidecar" for Microsoft Outlook that automates threat isolation while preserving privacy. It employs zero-disk forensic analysis and hardware-backed credential protection to serve users in high-risk environments without requiring Exchange Admin rights.

## 2. HARDENED ARCHITECTURE (REALITY)
- **UI:** Electron.js (Context-Isolated, Node-Integration Disabled).
- **Backend IPC:** Bidirectional Named Pipe (`\\.\pipe\microsoft_outlook_security_ipc`). Eliminates disk-based command triggers and race conditions.
- **Engine:** PowerShell Core / 5.1 with strict COM Scoping.
- **Credential Protection:** Windows DPAPI (via Electron safeStorage and PowerShell ProtectedData). API keys never exist in plaintext on disk or in command-line arguments.
- **Database:** Electron-store with Atomic File Locking (Rename-based) to prevent corruption during concurrent service/UI access.
- **Security:** Strict ACLs on `C:\ProgramData\MicrosoftOutlookSecurity` (System/Admin: Full; User: Read/Execute/Write-Data, but no Modify/Delete on core logic).

## 3. ZERO-DISK FORENSIC PROTOCOL
- **MIME Isolation:** Attachments are streamed directly from the COM layer into RAM byte arrays for hashing (SHA256/MD5).
- **No Temp Files:** Eliminates the use of `$env:TEMP` for forensic analysis, preventing EDR/AV interference and leaving no forensic footprint on the host disk.
- **Header Parsing:** Multi-hop `Received` chain analysis for true Originating IP extraction, bypassing common header-poisoning techniques.

## 4. RESOURCE & STABILITY MANAGEMENT
- **COM Lifecycle:** Strict `Marshal.ReleaseComObject` protocol for every reference (Folders, Items, PropertyAccessors) to prevent `OUTLOOK.EXE` zombie processes.
- **Parent Watchdog:** The Engine monitors the Electron Parent PID; immediate graceful exit and session cleanup occur if the parent process terminates.
- **Watchdog Timer:** 60-minute hard-kill for hung scanning processes to ensure system responsiveness under high load.

## 5. SCANNING & PRIORITIZATION
- **OnAccess:** Monitors [Unread] items. Uses a high-speed `HashSet` of `EntryIDs` to skip previously audited items.
- **3-Tier Failover:** 
    1. VirusTotal API (Primary).
    2. Team Cymru MHR (DNS-based fallback).
    3. Local Heuristics/Spam Rubrics (Final).
- **Spam Rubrics:** RFC-compliant SPF/DKIM validation and IP-to-Domain reverse DNS verification.

## 6. VERDICT ROUTING
- **MALICIOUS:** Confirmed Malware/Phishing -> **Deleted Items** (Isolation).
- **SPAM:** High-score suspicion -> **Junk Email** (Quarantine).
- **SAFE:** Whitelisted or Low-score -> **Inbox** (Untouched).

## 7. FUTURE VISION (PROPOSED)
- **Network Kill-Switch:** Immediate NIC disablement upon high-confidence Ransomware detection.
- **Heuristic Evolution:** Transition from static keyword matching to local LLM-based intent analysis for Phishing detection.
- **Credential Vault:** Integration with Windows Credential Manager (Vault) for all third-party SOC integrations.
- **Multi-Account:** Support for multiple MAPI profiles and Shared Mailboxes.
